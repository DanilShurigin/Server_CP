Подробное описание

Класс для записи информации в журнал работы сервера.

Пример использования класса для записи критической ошибки

try {
// Какой-то код, приводящий к критической ошибке
} catch(const ServerException& e) {
logger(e.what(), Fatal);
exit(EXIT_FAILURE);
}

LogPriority  priority

Запись в журнал работы.

Записывает только сообщения, уровень которых выше или равен установленному минимуму.

Аргументы
[in]messageЗаписываемое сообщение (строка)
[in]priorityУровень записи.
Возвращает
true Сообщение успешно записано, или запись отклонена из-за низкого уровня.
false Сообщение не записано из-за ошибки открытия файла.
